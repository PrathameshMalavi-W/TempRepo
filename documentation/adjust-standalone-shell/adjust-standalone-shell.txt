V1-Main

[#adjust-standalone-mode]
=== Adjust Standalone Mode

In OneCX v6, the @onecx/standalone-shell library has been renamed to @onecx/angular-standalone-shell.
All references must be updated accordingly.

Additionally, starting with v6, standalone applications must explicitly register standalone providers using provideStandaloneProviders().

==== Required changes

Update all imports from @onecx/standalone-shell to @onecx/angular-standalone-shell

Update the dependency name in package.json

Update any webpack configuration referencing the old package

Register provideStandaloneProviders() in the standalone module providers

==== Update package dependency

Replace the old package with the new one.

[source,bash]
npm uninstall --save @onecx/standalone-shell
npm install --save @onecx/angular-standalone-shell

==== Update module imports and providers

Import StandaloneShellModule and provideStandaloneProviders from the new package and register the providers explicitly.

Example: updated-app-standalone.module.ts

.Before
[source,typescript]

import { StandaloneShellModule } from '@onecx/standalone-shell';

@NgModule({
imports: [StandaloneShellModule]
})
export class AppModule {}
.After
[source,typescript]

import {
StandaloneShellModule,
provideStandaloneProviders
} from '@onecx/angular-standalone-shell';

@NgModule({
imports: [StandaloneShellModule],
providers: [
provideStandaloneProviders()
]
})
export class AppModule {}

==== Webpack configuration

If the standalone shell package is referenced in a custom webpack configuration, update the package name to @onecx/angular-standalone-shell.

==== Result

After applying these changes, the standalone application is compatible with OneCX v6 and correctly initializes standalone-specific providers.
